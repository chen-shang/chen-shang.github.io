SpringBoot中HandlerInterceptor注入其他Service报错循环依赖问题

原创 JAVA SpringBoot
JAVA SpringBoot
发布日期:   2023-11-29
更新日期:   2023-11-29
文章字数:   3k

Java 写的 HandlerInterceptor 代码结构如下

public class CollectionSystemInterceptor implements HandlerInterceptor {
    //注意这里的引入方式
    @Autowired
    private IDmCollectionAgencyInnerApiRpc dmCollectionAgencyInnerApiRpc;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        // 从Header中获取 催收机构编码
        String collectionCode = request.getHeader("HEADER_COLLECTION_CODE");
        Preconditions.checkArgument(StringUtils.isNotBlank(collectionCode), "HEADER_COLLECTION_CODE in header can not be null");
        dmCollectionAgencyInnerApiRpc.doSomething()
        return HandlerInterceptor.super.preHandle(request, response, handler);
    }
}

关键报错信息

Caused by: org.springframework.beans.factory.BeanCurrentlyInCreationException: Error creating bean with name 'mvcResourceUrlProvider': Requested bean is currently in creation: Is there an unresolvable circular reference?

解决方案
试过了好几种都不管用，最后使用 @Lazy 的方案可行，特此记录一下

public class CollectionSystemInterceptor implements HandlerInterceptor {
    //注意这里的引入方式,这里添加了@Lazy注解
    @Autowired
    @Lazy
    private IDmCollectionAgencyInnerApiRpc dmCollectionAgencyInnerApiRpc;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        // 从Header中获取 催收机构编码
        String collectionCode = request.getHeader("HEADER_COLLECTION_CODE");
        Preconditions.checkArgument(StringUtils.isNotBlank(collectionCode), "HEADER_COLLECTION_CODE in header can not be null");
        dmCollectionAgencyInnerApiRpc.doSomething()
        return HandlerInterceptor.super.preHandle(request, response, handler);
    }
}
