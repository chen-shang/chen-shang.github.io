这篇文档记录了我在 GitHub 上收藏的项目，涵盖了 AI Agent、PDF 处理、文档转换、画图工具、搜索等多个领域。

1. AI / Agent 相关

1.1 anthropics/skills

项目 说明
链接 https://github.com/anthropics/skills
标签 agent-skills
简介 Public repository for Agent Skills

用途：这是 Anthropic 官方维护的 Agent Skills 仓库，提供了各种 Claude Code 可用的技能（Skills）。如果你想扩展 Claude Code 的能力，这个仓库是官方的技能库来源。

1.2 affaan-m/everything-claude-code

项目 说明
链接 https://github.com/affaan-m/everything-claude-code
标签 ai-agents, anthropic, claude, claude-code, developer-tools, llm, mcp, productivity
简介 The agent harness performance optimization system. Skills, instincts, memory, security, and research-first development for Claude Code, Codex, Opencode, Cursor and beyond.

用途：这是一个全面的 Claude Code 性能优化系统，包含了：

  • Skills（技能）：扩展 Claude Code 能力的模块
  • Instincts（本能）：内置的行为模式
  • Memory（记忆）：知识库和记忆系统
  • Security（安全）：安全相关的配置
  • Research-first development：研究优先的开发方法

适合想要深度定制和优化 Claude Code 工作流的开发者。

1.3 QuantumNous/new-api

项目 说明
链接 https://github.com/QuantumNous/new-api
标签 ai-gateway, claude, deepseek, gemini, newapi, openai, rerank
简介 A unified AI model hub for aggregation & distribution. It supports cross-converting various LLMs into OpenAI-compatible, Claude-compatible, or Gemini-compatible formats. A centralized gateway for personal and enterprise model management. 🍥

用途：统一的大模型聚合与分发平台，主要功能：

  • 多模型支持：支持 OpenAI、Claude、Gemini、DeepSeek 等多种大模型
  • 协议转换：可以将不同模型的 API 转换为统一格式（OpenAI 兼容、Claude 兼容、Gemini 兼容）
  • 网关功能：企业和个人可以将其作为统一的模型管理入口
  • Rerank：支持 rerank 功能，提升搜索效果

适合需要管理多个大模型 API 的开发者或企业。

1.4 ModelEngine-Group/nexent

项目 说明
链接 https://github.com/ModelEngine-Group/nexent
标签 agent, agentic-ai, agentic-framework, agentic-rag, agentic-workflow, ai, harness, harness-engineering, llm, mcp, multi-agent, rag
简介 Nexent is a zero-code platform for auto-generating production-grade AI agents using Harness Engineering principles — unified tools, skills, memory, and orchestration with built-in constraints, feedback loops, and control planes.

用途：零代码平台，用于自动生成生产级 AI Agent，核心特性：

  • 零代码：无需编程即可创建 AI Agent
  • Harness Engineering：使用 Harness 工程化原则
  • 统一工具：统一的工具、技能、记忆系统
  • 工作流编排：内置编排能力和控制平面
  • Agentic RAG：支持 RAG（检索增强生成）
  • 多 Agent：支持多 Agent 协作

适合非技术用户或想要快速构建 AI Agent 的团队。

1.5 jd-opensource/joyagent-jdgenie

项目 说明
链接 https://github.com/jd-opensource/joyagent-jdgenie
简介 开源的端到端产品级通用智能体

用途：京东开源的端到端产品级通用智能体，是一个生产环境可用的通用 AI Agent 框架。

1.6 farouqaldori/claude-island

项目 说明
链接 https://github.com/farouqaldori/claude-island
简介 Claude Code notifications without the context switch. A minimal, always-present session manager for macOS.

用途：MacOS 上的 Claude Code 会话管理器，提供：

  • 无需切换上下文：始终保持 Claude Code 通知可见
  • 轻量级：极简的 macOS 原生应用
  • 会话管理：帮助你管理多个 Claude Code 会话

适合需要在 Mac 上频繁使用 Claude Code 的开发者。

1.7 GitFrog1111/badclaude

项目 说明
链接 https://github.com/GitFrog1111/badclaude

用途：Claude Code 相关项目（具体功能未公开披露）。

2. PDF 工具

2.1 PDFCraftTool/pdfcraft

项目 说明
链接 https://github.com/PDFCraftTool/pdfcraft
简介 PDFCraft is a free, privacy-focused PDF toolkit that runs entirely in your browser. With 90+ professional tools, you can edit, convert, merge, split, and secure your PDF files without ever uploading them to a server.

用途：免费的隐私优先 PDF 工具箱，核心特性：

  • 90+ 专业工具：编辑、转换、合并、拆分、保护 PDF
  • 本地运行：完全在浏览器中运行，不上传到服务器
  • 隐私保护：所有处理都在本地完成，敏感文档更安全
  • 免费使用：无需付费即可使用全部功能

适合需要处理 PDF 但注重隐私的用户。

3. 文档转换

3.1 hanshuaikang/AI-Media2Doc

项目 说明
链接 https://github.com/hanshuaikang/AI-Media2Doc
标签 ai, bilibili, chatgpt, openai, python, subtitles-generator, vue, xiaohongshu, youtube
简介 一键将音视频转化为小红书/公众号/知识笔记/思维导图/视频字幕等各种风格的文档。

用途：AI 驱动的音视频转文档工具，主要功能：

  • 音视频转文档：将音视频内容转换为文字文档
  • 多平台支持：支持 YouTube、Bilibili 等平台
  • 多风格输出：小红书、公众号、知识笔记、思维导图、视频字幕等
  • AI 优化：使用 ChatGPT / OpenAI 进行智能优化

适合内容创作者、教育工作者、媒体从业者。

4. 画图 + AI

项目 说明
链接 https://github.com/DayuanJiang/next-ai-draw-io

项目 说明

用途：next-ai-draw-io 的中文二次开发版，增强功能：

  • 国内大模型支持：七牛云、千问、GLM、豆包等

5. Elasticsearch 生态

5.1 elastic/elasticsearch

项目 说明
链接 https://github.com/elastic/elasticsearch
标签 elasticsearch, java, search-engine
简介 Free and Open Source, Distributed, RESTful Search Engine

用途：开源的分布式 RESTful 搜索引擎，核心特性：

  • 全文搜索：强大的全文搜索能力
  • 分布式：支持大规模分布式部署
  • RESTful API：统一的 RESTful 接口
  • Java 开发：基于 Java 的开源项目

Elasticsearch 是 ELK（Elasticsearch + Logstash + Kibana）栈的核心组件。

5.2 elastic/kibana

项目 说明
链接 https://github.com/elastic/kibana
标签 dashboards, elasticsearch, hacktoberfest, kibana, metrics, observability, visualizations
简介 Your window into all of your data

用途：Elasticsearch 的可视化管理和数据可视化平台，功能：

  • 仪表盘：创建交互式数据仪表盘
  • 可视化：丰富的图表和可视化组件
  • 可观测性：应用和系统可观测性工具
  • 指标分析：日志和指标分析

是 ELK 栈中负责数据可视化的组件，与 Elasticsearch 配合使用。
